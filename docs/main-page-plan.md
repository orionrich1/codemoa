# 메인 페이지 개선 기획서

## 개요

현재 CodeMoa 메인 페이지(`/`)는 10개 이상의 섹션이 수직 나열된 "정보 나열형" 구조다.
데이터는 풍부하지만 **사용자가 목적지에 빠르게 도달하기 어렵고**, 로그인 여부와 관계없이 거의 동일한 화면이 노출된다.
이 기획서는 **목적별 사용자 여정**을 기준으로 섹션 우선순위를 재정의하고, 로그인 사용자에게 개인화 경험을 제공하며, ElevenLabs 디자인 시스템(`DESIGN.md`) 기준으로 시각 일관성을 확보하는 방향으로 메인 페이지를 개선한다.

---

## 사용자 스토리

- **처음 방문한 비로그인 사용자**로서, CodeMoa가 어떤 서비스인지 5초 이내에 파악하고 회원가입·탐색 행동을 하고 싶다.
- **재방문 로그인 사용자**로서, 내가 놓친 활동(미답변 Q&A, 마감 임박 공모전, 새 커뮤니티 글)을 한눈에 파악하고 싶다.
- **AI 문제 풀이 목적 사용자**로서, 메인에서 오늘 풀어야 할 문제로 바로 진입하고 싶다.
- **취업/팀 모집 목적 사용자**로서, 현재 올라와 있는 공고·모집 글 상태를 빠르게 스캔하고 싶다.

## 현재 구조의 핵심 문제

| 문제 영역 | 현재 상태 | 영향 |
|-----------|-----------|------|
| 섹션 우선순위 불명확 | 10개+ 섹션이 동일 비중으로 나열됨 | 사용자가 무엇부터 볼지 결정 불가 |
| 개인화 부재 | 히어로 외 로그인/비로그인 동일 화면 | 재방문 사용자 재참여 동기 약함 |
| 게시판별 최신 섹션 | 5컬럼 가로 나열 | 모바일에서 스캔 불가, 정보 밀도 낮음 |
| 섹션 간 시각 구분 약함 | 유사한 카드 스타일 반복 | 스크롤 피로, 섹션 인지 어려움 |
| 빈 상태 데모 데이터 | DB 미연결 시 더미 데이터 노출 | 실데이터 착각 가능, 신뢰도 저하 |
| 오늘의 AI 문제 위치 | 통계 바·칩 트레이 아래 묻힘 | 핵심 기능 진입점이 눈에 안 띔 |
| 디자인 시스템 미적용 | 메인 전용 스타일이 공통 토큰과 혼재 | 브랜드 일관성 약함 |

## 기능 상세

### 기능 1: 히어로 섹션 — 비로그인 랜딩 강화

현재 히어로는 카피 문구 + CTA 2개로 충분하지만, **서비스 가치를 즉각 전달하는 요소**가 부족하다.

- 수락 조건:
  - [ ] 헤드라인 1줄, 서브 텍스트 1줄, CTA Primary 1개(`AI 문제 풀기`), CTA Secondary 1개(`커뮤니티`) 구조 유지
  - [ ] 히어로 하단에 **3가지 핵심 가치**를 아이콘+텍스트로 한 줄 나열 (예: "AI 피드백", "개발자 커뮤니티", "채용 정보")
  - [ ] 회원가입 링크를 CTA 근처에 텍스트 링크로 추가 ("아직 계정이 없으신가요? 가입하기")
  - [ ] 통합 검색창은 히어로 내 중요도를 낮추거나 헤더로 이전 (별도 문서로 분리 가능)
  - [ ] CTA Primary는 검정 배경·흰 글씨·완전 둥근 알약형, CTA Secondary는 웜스톤(연한 베이지) 배경·검정 글씨 알약형으로 표시되어야 함

---

### 기능 2: 로그인 사용자 개인화 요약 카드

현재 로그인 히어로는 닉네임·등급·포인트를 보여주지만, **"지금 내가 해야 할 것"을 안내하지 않는다**.

- 수락 조건:
  - [ ] 로그인 시 히어로 아래에 **"내 활동 요약" 카드** 1줄 노출 (최대 3개 항목)
  - [ ] 항목 후보 (우선순위 순):
    1. 마지막으로 시도한 문제가 있을 경우 → "미완료 문제: [제목] 이어 풀기" 링크
    2. 이번 주 푼 문제가 0개일 경우 → "이번 주 첫 문제 도전하기" CTA
    3. 내 Q&A에 새 답변이 달렸을 경우 → "내 질문에 답변이 달렸습니다" 링크
  - [ ] 데이터가 없는 항목은 노출하지 않음 (항목이 0개면 카드 자체를 숨김)
  - [ ] 이 카드는 서버 렌더링 (JS 없이 Thymeleaf 조건부 출력)

---

### 기능 3: 섹션 순서 재정의

현재 섹션 순서를 **사용자 행동 빈도 및 전환 중요도** 기준으로 재배치한다.

**개선 전 순서:**
히어로 → 통계 바 → 칩 트레이 → 오늘의 AI 문제 → 핵심 허브(4개 아이콘) → 커뮤니티 피드+사이드 → 게시판별 최신 → IT 정보 허브 → 프로젝트 다이어리 → 채용·팀 → FAQ

**개선 후 순서:**

| 순서 | 섹션명 | 변경 사항 |
|------|--------|-----------|
| 1 | 히어로 | 유지 (기능 1 적용) |
| 2 | 개인화 요약 카드 | 신규 (로그인 시만) |
| 3 | 통계 바 | 유지 |
| 4 | 오늘의 AI 문제 | **위로 이동** (칩 트레이 앞으로) |
| 5 | 칩 트레이 | 기존 위치보다 아래로 |
| 6 | 커뮤니티 피드 + 사이드바 | 유지 |
| 7 | IT 정보 허브 (강좌·도서·공모전) | 유지 |
| 8 | 채용·팀 모집 | 유지 |
| 9 | 게시판별 최신 | **아래로 이동** (탭형 개선 후) |
| 10 | 프로젝트 다이어리 | 유지 (위치 검토) |
| 11 | FAQ | 유지 |
| (제거) | 핵심 허브 4아이콘 그리드 | 칩 트레이와 역할 중복 → 제거 |

- 수락 조건:
  - [ ] "핵심 허브" 4아이콘 섹션은 칩 트레이와 중복이므로 **제거**
  - [ ] "오늘의 AI 문제" 위젯이 히어로·개인화 요약 카드 다음으로 노출됨
  - [ ] 섹션 간 시각적 구분(배경색 교차 또는 구분선)이 적용됨

---

### 기능 4: 게시판별 최신 — 탭형 UX로 전환

현재 5컬럼 가로 카드는 데스크톱에서도 정보 밀도가 낮고, 모바일에서는 세로 스택으로 뭉개진다.

- 수락 조건:
  - [ ] 게시판 탭(자유 / Java / JavaScript / Python / Kotlin)을 상단에 배치
  - [ ] 탭 전환 시 해당 게시판 최신 글 **최대 8건** 리스트로 표시
  - [ ] 각 리스트 아이템은 **제목 + 날짜 + 추천수** 3가지 메타 표시
  - [ ] 현재 탭에서 "더보기" 클릭 시 해당 게시판으로 이동
  - [ ] 모바일에서 탭이 가로 스크롤 가능해야 함

---

### 기능 5: 빈 상태(Empty State) 정책 정비

현재 DB가 비어있으면 더미 데이터를 채운다. 이는 **실제 데이터처럼 오인**될 수 있다.

- 수락 조건:
  - [ ] 더미 데이터를 사용하는 섹션(랭킹, 강좌, 도서, 공모전, 채용, 팀 모집)에 "샘플" 표시 또는 **더미 데이터 제거** 후 빈 상태 UX로 대체 (개발자 협의 후 결정)
  - [ ] 빈 상태 UX 기준: 아이콘 1개 + 안내 문구 1줄 + 행동 유도 링크(CTA) 1개
  - [ ] `mainShowDemoNotice` 배너는 커뮤니티 글 없을 때만 표시, 위치를 해당 섹션 안으로 이동

---

### 기능 6: Q&A 하이라이트 개선

현재 Q&A 하이라이트는 단순 제목 목록이다. **"지금 답해줄 수 있는 질문"** 관점으로 개선한다.

- 수락 조건:
  - [ ] 미답변 Q&A를 우선 노출 (답변완료 글은 하단 배치 또는 제외)
  - [ ] 각 Q&A 아이템에 **경과 시간** 표시 (예: "3시간 전", "2일 전")
  - [ ] "미답변 [n]건" 뱃지를 섹션 헤더에 표시
  - [ ] 리스트 하단에 "질문하기" CTA 링크 추가

---

### 기능 7: 통계 바 확장 (선택)

현재 4개 항목에 이번 주 활동 지표를 추가해 서비스의 활발함을 전달한다.

- 수락 조건:
  - [ ] 기존 4개 항목(총 회원, 등록 문제, 총 게시글, 공모전) 유지
  - [ ] 추가 후보 1개: **"이번 주 활동자 n명"** 또는 **"오늘 새 글 n건"** (데이터 가용 시)
  - [ ] 모바일에서 항목이 2×2 그리드 또는 가로 스크롤로 표시되어야 함

---

### 기능 8: 디자인 시스템 적합성 — 메인 페이지 전용

메인 페이지의 시각 요소를 `DESIGN.md`(ElevenLabs 디자인 시스템) 기준에 맞춰 통일한다.
구체적인 색상·수치는 `DESIGN.md`를 참고하며, 아래는 **사용자가 체감할 수 있는 요구사항**이다.

- 수락 조건:
  - [ ] 메인 페이지 색상은 화이트, 라이트 그레이, 웜스톤(연한 베이지), 블랙 계열만 사용 — 기존 브랜드 틸/액센트 색은 메인에서 미노출
  - [ ] 섹션 제목은 얇은(light) 두께의 폰트로, 굵게 처리하지 않음
  - [ ] 카드/패널은 흰 배경, 둥근 모서리, 그림자 3겹 스택 형태로 통일
  - [ ] 채용·팀 모집 섹션은 웜스톤 배경의 독립 구역으로 시각적으로 구분됨
  - [ ] 모든 버튼·칩의 터치 영역은 세로 44px 이상 확보 (모바일 접근성)
  - [ ] 키보드 포커스 시 각 링크·버튼에 연한 파란색 포커스 링이 표시됨
  - [ ] 리스트 행 높이는 최소 48px 이상 유지 (터치 친화적)

## 시나리오

### 정상 케이스

**A. 비로그인 첫 방문:**
1. `/` 접속 → 히어로(카피 + CTA 2개 + 서비스 가치 3줄) 노출
2. 통계 바로 서비스 규모 확인
3. 오늘의 AI 문제 위젯으로 핵심 기능 체험 유도
4. 커뮤니티 피드에서 최신 글 훑어보기
5. IT 정보 허브에서 강좌/공모전 탐색
6. 채용·팀 섹션에서 공고 확인
7. FAQ로 서비스 이해 → 가입 결정

**B. 로그인 재방문:**
1. `/` 접속 → 히어로(닉네임·등급·포인트) + **개인화 요약 카드** 노출
2. "미완료 문제 이어 풀기" 또는 "이번 주 첫 문제" CTA 클릭 → 문제 페이지 이동
3. 커뮤니티 피드 인기/최신 탭으로 관심 글 탐색
4. 사이드바 Q&A 하이라이트에서 미답변 질문에 답변 참여

### 예외 케이스

1. **DB 전체 비어 있음**: 각 섹션이 "빈 상태 UX"를 노출하고, 관리자만 볼 수 있는 데모 배너로 구분
2. **오늘의 AI 문제 없음**: 위젯 자체를 숨김
3. **개인화 요약 카드 데이터 없음**: 카드 자체 숨김, 히어로 바로 아래 통계 바로 넘어감
4. **모바일(375px) 접속**: 5컬럼 게시판 섹션이 탭형으로 표시, 통계 바 항목이 2×2로 재배열
5. **로그인 세션 만료 상태**: 개인화 카드 미노출 (서버 렌더링이므로 자동 처리)

## 데이터 요구사항

| 필드명 | 설명 | 신규 여부 | 필수 |
|--------|------|-----------|------|
| `personalizedActions` | 개인화 요약 카드 항목 목록 (최대 3개) | 신규 | 로그인 시 |
| `lastAttemptedProblem` | 마지막으로 시도한 문제 (제목, ID) | 신규 | 선택 |
| `weeklyProblemCount` | 이번 주 푼 문제 수 | 신규 | 로그인 시 |
| `myUnansweredQnaCount` | 내 Q&A 중 미답변 건수 | 신규 | 로그인 시 |
| `statsActiveUsersThisWeek` | 이번 주 활동 사용자 수 | 신규 | 선택 |
| `highlightQnas` | Q&A 하이라이트 목록 — 경과 시간 필드 추가 | 기존 확장 | 필수 |
| `popularPosts` | 인기 커뮤니티 글 | 기존 유지 | 필수 |
| `weeklyRanking` | 주간 랭킹 TOP 5 | 기존 유지 | 필수 |

> 신규 필드 중 로그인 시 필요한 항목은 비로그인 시 null로 설정한다.
