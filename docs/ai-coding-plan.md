# AI 코딩 문제 풀이 기능 개선 기획서

## 개요

현재 AI 코딩 문제 풀이 기능(`/problems`)은 **문제 목록 → 답안 입력 → AI 채점 결과** 흐름의 기본 뼈대는 갖추었으나,
풀이 이력이 저장되지 않고, 코드 입력 환경이 일반 텍스트박스 수준이며, 결과 페이지 UI가 미완성(`임시 css 사용중`)인 상태다.
이 기획서는 **사용자가 문제를 반복해서 풀고, 성장을 체감하며, AI를 자연스럽게 학습 도구로 활용**할 수 있도록 기능 추가와 고도화 방향을 정의한다.

---

## 사용자 스토리

- **취업 준비 대학생**으로서, 내가 과거에 푼 문제와 받은 점수를 확인하고 성장을 느끼고 싶다.
- **코딩 학습자**로서, 문제를 틀렸을 때 어디가 부족했는지 AI에게 구체적인 피드백을 받고 싶다.
- **반복 학습자**로서, 틀린 문제만 모아서 다시 풀고 싶다.
- **AI 활용 학습자**로서, 문제를 풀다가 막히면 코드 작성 화면을 벗어나지 않고 AI에게 바로 질문하고 싶다.
- **목표 지향 학습자**로서, 내가 어떤 언어와 난이도를 많이 틀리는지 통계로 보고 싶다.

## 현재 구조의 핵심 문제

| 문제 영역 | 현재 상태 | 사용자 불편 |
|-----------|-----------|-------------|
| 풀이 이력 없음 | 제출 후 저장 안 됨 | 지난 점수·피드백을 다시 볼 수 없음 |
| 코드 입력 환경 열악 | 일반 textarea | 탭 키, 들여쓰기, 문법 색상 없음 |
| 결과 페이지 UI 미완성 | 임시 CSS, 구조 없음 | 점수·피드백 가독성 매우 낮음 |
| AI 질문이 문제와 분리 | `/problems/aiAnswer` 별도 페이지 | 문제 풀다 AI 질문 시 화면 이탈 필요 |
| 푼 문제 구분 없음 | 목록에서 풀었는지 알 수 없음 | 안 푼 문제를 찾기 위해 전부 클릭해야 함 |
| Kotlin 언어 없음 | JAVA·JS·Python 3개만 | 커뮤니티 Kotlin 게시판과 불일치 |
| 포인트 지급 기준 불명확 | 제출 시 포인트 지급 로직 미구현 | 포인트 시스템 동기 부여 기능 사라짐 |
| 페이지네이션·검색 없음 | 문제 수 증가 시 전체 나열 | 문제가 많아지면 탐색 불가 |

## 화면 목록

| URL | 화면명 | 설명 |
|-----|--------|------|
| `/problems/` | 문제 목록 | 필터·검색·풀이 상태 표시 |
| `/problems/problemDetail?problemId={id}` | 문제 풀기 | 코드 입력 + 인라인 AI 질문 |
| `/problems/problemResult` | AI 채점 결과 | 점수·피드백·다시 풀기 |
| `/problems/myHistory` | 내 풀이 이력 | 과거 제출 목록·점수 |
| `/problems/stats` | 내 학습 통계 | 언어별·난이도별 정답률 |
| `/problems/aiAnswer` | AI 자유 질문 | 문제와 무관한 코딩 질문 (기존 유지) |
| `/problems/problemWrite` | 문제 작성 (관리자) | 기존 유지 + 미리보기 추가 |

## 기능 상세

### 기능 1: 풀이 이력 저장

현재 제출 후 결과가 저장되지 않아, 과거 피드백을 다시 볼 수 없고 다시 풀기도 불가능하다.

- 수락 조건:
  - [ ] 로그인 사용자가 문제를 제출하면 **제출 이력 1건**이 저장됨
  - [ ] 저장 항목: 사용자 ID, 문제 ID, 제출 답안, AI 점수(0~100), 제출 일시
  - [ ] 같은 문제를 여러 번 제출할 수 있으며, **각 제출마다 이력이 누적**됨 (덮어쓰기 금지)
  - [ ] 비로그인 사용자는 제출 시 로그인 유도 안내를 표시하고 제출을 차단함
  - [ ] 저장된 이력은 "내 풀이 이력" 페이지에서 확인 가능

---

### 기능 2: 문제 목록 — 풀이 상태 및 탐색성 개선

현재 목록은 단순 나열이며, 내가 풀었는지 여부와 점수를 알 수 없다.

- 수락 조건:
  - [ ] 로그인 사용자에게 각 문제 아이템에 **풀이 상태 배지** 표시
    - 미시도: 표시 없음 (기본)
    - 풀었음 + 최고 점수: 예) ✓ `82점` 뱃지
  - [ ] 문제 제목으로 **키워드 검색** 가능 (입력 즉시 클라이언트 측 필터링)
  - [ ] 기존 언어 필터(JAVA·JS·Python)에 **Kotlin** 추가
  - [ ] 문제가 20개 초과 시 **페이지네이션** 또는 무한 스크롤 적용
  - [ ] 목록 상단에 **내 진행 요약** 1줄 표시: "전체 n개 중 m개 풀었습니다 (정답률 k%)"
    → 비로그인 시 미표시

---

### 기능 3: 문제 풀기 화면 — 코드 입력 환경 개선

현재 답안 입력이 일반 textarea여서 코딩 경험이 나쁘다.

- 수락 조건:
  - [ ] 코드 입력 영역에서 **Tab 키가 들여쓰기(스페이스 4칸)로 동작** (현재 포커스 이탈)
  - [ ] 코드 입력 영역은 **줄 번호** 가 표시되어야 함
  - [ ] 언어 선택 드롭다운이 코드 입력 영역 상단에 표시되어, 선택한 언어에 맞게 힌트 텍스트가 바뀜 (예: "Java 코드를 입력하세요")
  - [ ] 문제 내용 패널과 코드 입력 패널의 **세로 높이를 브라우저 뷰포트에 맞게 고정** (스크롤이 카드 내부에서 발생)
  - [ ] 모바일(768px 이하)에서 문제 내용 패널이 **접기/펼치기** 가능

---

### 기능 4: 문제 풀기 화면 — 인라인 AI 질문 패널

현재 AI 질문을 하려면 `/problems/aiAnswer`로 이동해야 해서 작성 중인 코드가 사라진다.

- 수락 조건:
  - [ ] 문제 풀기 화면 우측 하단에 **"AI에게 질문"** 버튼 상시 표시
  - [ ] 버튼 클릭 시 화면 하단에서 **슬라이드업 패널** 또는 **모달**로 AI 질문 입력창 열림
  - [ ] 질문 전송 시, 현재 문제의 제목·언어 정보가 AI 질문 컨텍스트에 자동 포함됨
  - [ ] AI 답변은 패널 안에서 표시되며, 패널을 닫아도 **코드 입력 내용이 유지**됨
  - [ ] 패널 내 AI 질문은 세션 동안 최대 5회까지 허용 (남은 횟수 표시)

---

### 기능 5: AI 채점 결과 페이지 재설계

현재 결과 페이지는 UI가 "임시 CSS" 상태로 미완성이다.

- 수락 조건:
  - [ ] 페이지 상단에 **점수를 크게 표시** (예: `82 / 100`)
  - [ ] 점수 아래에 **등급 표현** 표시: 90점↑ 우수, 70점↑ 양호, 50점↑ 보통, 50점↓ 재도전
  - [ ] 문제 내용 / 제출 답안 / AI 피드백 3개 영역을 **탭** 또는 **아코디언**으로 구분
  - [ ] 모범 답안은 **기본 숨김 상태**, "모범 답안 보기" 버튼 클릭 시 표시
  - [ ] 하단 버튼: "다시 풀기"(문제 페이지로 이동) / "다른 문제 풀기"(목록) / "내 이력 보기"
  - [ ] AI 피드백 텍스트에서 **코드 블록(`\`\`\`...`)이 구분되어 표시**됨

---

### 기능 6: 내 풀이 이력 페이지 (신규)

과거에 푼 문제와 점수를 한눈에 볼 수 있는 개인 이력 페이지를 제공한다.

- 수락 조건:
  - [ ] 로그인 사용자만 접근 가능; 비로그인 시 로그인 페이지로 리다이렉트
  - [ ] 내가 제출한 이력을 **최신 순**으로 목록 표시
  - [ ] 각 이력 아이템: 문제 제목, 언어, 난이도, 점수, 제출 일시
  - [ ] 같은 문제를 여러 번 푼 경우 **최고 점수**를 강조 표시
  - [ ] 점수 필터: 전체 / 90점↑ / 50점↓ (재도전 필요)
  - [ ] 목록에서 해당 문제로 바로 이동하는 "다시 풀기" 링크 제공
  - [ ] 이력이 없으면 "아직 푼 문제가 없습니다. 첫 문제 도전하기" 빈 상태 표시

---

### 기능 7: 내 학습 통계 페이지 (신규)

언어별·난이도별 약점을 시각적으로 파악할 수 있는 통계 화면을 제공한다.

- 수락 조건:
  - [ ] 로그인 사용자만 접근 가능
  - [ ] **총 제출 수 / 평균 점수 / 최고 점수** 숫자 3개를 상단 카드로 표시
  - [ ] **언어별 평균 점수**: JAVA / JavaScript / Python / Kotlin 각각의 평균 (막대 또는 수치)
  - [ ] **난이도별 정답률**: 상·중·하 각각 "시도 n회 중 m회 70점↑"로 표시
  - [ ] **최근 7일 제출 수** 표시 (연속 학습 현황)
  - [ ] 통계 데이터가 없으면 (이력 0건) 빈 상태 + "문제 풀러 가기" CTA 표시

---

### 기능 8: 포인트 지급 정책 연동

현재 문제 제출 시 포인트 지급 로직이 없어 포인트 시스템 동기 부여 효과가 없다.

- 수락 조건:
  - [ ] 문제 제출 후 AI 점수에 따라 **포인트 자동 지급**
    - 90점 이상: 100P
    - 70~89점: 60P
    - 50~69점: 30P
    - 50점 미만: 10P (도전 자체에 보상)
  - [ ] 같은 문제는 **하루 1회만 포인트 지급** (반복 제출로 포인트 남용 방지)
  - [ ] 결과 페이지에서 **"+ n 포인트 획득"** 메시지 표시
  - [ ] 포인트 지급 여부(오늘 이미 받음)도 결과 페이지에서 안내

---

### 기능 9: 문제 작성 폼 개선 (관리자)

관리자가 문제를 작성할 때 실제 표시 형태를 미리 확인할 수 없다.

- 수락 조건:
  - [ ] 문제 내용 입력 영역 우측 또는 하단에 **실시간 미리보기** 패널 제공
  - [ ] 힌트 필드에 **"없음"** 기본값 또는 선택 입력 안내 표시 (현재 빈 칸이면 힌트 버튼이 빈 내용을 표시)
  - [ ] 언어(category) 선택을 자유 입력 → **드롭다운 선택**으로 변경 (JAVA·JavaScript·Python·Kotlin)
  - [ ] 난이도(difficulty) 선택을 자유 입력 → **드롭다운 선택**으로 변경 (상·중·하)
  - [ ] 작성 완료 후 **"작성한 문제 바로 보기"** 링크 제공

## 시나리오

### 정상 케이스

**A. 처음 방문 → 문제 풀기 → 결과 확인:**
1. `/problems/` 접속 → 언어·난이도 필터로 원하는 문제 탐색
2. 문제 아이템 클릭 → 문제 풀기 화면 진입
3. 코드 입력 → 막히면 "AI에게 질문" 패널 열기 → 힌트 얻음 → 패널 닫기 → 코드 계속 작성
4. 제출 클릭 → 결과 페이지로 이동 → 점수·피드백 확인
5. 포인트 획득 메시지 확인
6. "다시 풀기" 또는 "다른 문제 풀기" 선택

**B. 재방문 → 약점 파악 → 재도전:**
1. `/problems/stats` 접속 → JavaScript 평균 점수가 낮음을 확인
2. `/problems/` 에서 JavaScript 필터 선택 → 목록에서 낮은 점수 문제 찾기
3. `/problems/myHistory` 에서 50점 이하 필터 → "다시 풀기" 링크 클릭
4. 재도전 후 점수 향상 확인

### 예외 케이스

1. **비로그인 제출 시도**: 제출 버튼 클릭 시 "로그인이 필요합니다" 안내 + 로그인 페이지 링크
2. **AI 서버 응답 지연/실패**: 결과 페이지에서 "AI 피드백을 가져오는 중입니다" → 30초 초과 시 "잠시 후 다시 시도해 주세요" 안내; 점수는 0점으로 저장하지 않고 이력 저장 보류
3. **빈 코드 제출**: 제출 전 클라이언트 검증 — 빈 입력 시 "답안을 입력해 주세요" 오류 메시지 표시
4. **문제가 없는 경우**: 필터 조합 결과 0건일 때 "해당 조건의 문제가 없습니다. 필터를 초기화하세요" 안내
5. **하루 포인트 중복 지급 시도**: 결과 페이지에서 "오늘 이미 이 문제로 포인트를 받으셨습니다" 안내, 포인트 미지급
6. **인라인 AI 질문 5회 초과**: "오늘 AI 질문 횟수를 모두 사용했습니다. 내일 다시 이용해 주세요" 안내
